游行

           遭遇套路后，我就开始准备维权。打过12315和12345投诉，也上过消委会小程序投诉。调解不成就准备起诉了，网上查过很多资料，开始发现这是一个套路。我需要事件扩大因为这样才能增加曝光，才找到其他的“被害者”，至少在上牌那天就已经遇到了一个同在绿色出行买车上牌的家伙，我估计肯定有不少受害人。在前几次去白云公安局报警又不受理，在网上又恰巧看到一些游行的案例。既然游行可以增加一点可以加快处理案件的希望那就试一试，就网上查找了一些文书格式，准备设备和资料然后去白云公安那边交游行申请书，因为看到过一些没有经过允许就举牌拿喇叭上路逛的案例，都被请去喝茶了，既然已经知道这种行为是需要申请的，那就按规矩来吧！不然维权没搞成反而被请去喝茶就得不偿失了！

     申请游行需要游行申请书，相关的格式和资料网上都能找到这里就不细说了。早在2020年9月24日就去过公安局了，反馈一直到2021年1月22日才出，而且结果都直接寄回我老家的，都不是寄我住广州这里的地址。

宝塔面板安装记录

购买了一个轻量服务器，像我这种对Linux不熟悉的还是安装一个图形化管理面板好管理一点，运营服务器使用宝塔面板图形化管理。

使用ssh连接服务器

可以使用网页的SSH连接服务器

或者使用软件进行连接（如Xshenll 7），在命令行输入从宝塔Linux安装页面复制的安装脚本命令

yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh

在 Do you want to install Bt-Panel to the /www directory now?(y/n): 按键盘 y

安装完成后会给一个面板地址，需要在云服务器管理控制台打开相应的端口 8888 进入控制台找到防火墙/安全组 添加一个TCP 端口为8888 的端口

完成后打开命令行给的外网地址,外网面板地址是类似这样的: http://xx.xx.xx.xx.:8888/xxxxxx，打开后进去后就是面板主页了，

为了服务器安全！！！ 切记要修改 面板端口，安全入口，面板用户，面板密码，

给面板加上域名解析

进入域名DNS解析管理，添加一个记录类型为 A 记录值 xxxx.xxxx.xxxx.xxxx (服务器ipV4的地址)

这是给“相关人员”查看的一篇文章!

如题，这篇文章主要是给“相关人员”看的。因为下面的内容写的都是一些关于站长，向“相关人员”反映一些事情、表达我的一些意愿。

我经历的这次遭遇，找工作被骗、报警不归管辖、依法提交了两次游行申请才出书面告知（还是延迟版的）、诉讼又不理我的证据，最终无一例外就是败诉。而处理这一件事因管辖地问题，要分别在管辖地立案。比如“广州市白云区人民法院”案号：（2020）粤0111民初8888号、“珠海市香洲区人民法院”案号：（2020）粤0402民初9146号、“上海市虹口区人民法院”案号： （2020）沪0109民初18562号。

那个（2020）沪0109民初18562号一案的判决一样是败诉，我认为审理案件的法官同样也是没看证据的，因为签合同时是让对方解释了内容才签名的，从微信聊天记录、通话记录中都能分析出来。

我真是太难了！或许会有人说：”我凭什么相信你 ，而不相信官方的’结果’，你比官方更权威吗？”对于这种质疑我可以保证，我在本站上发布的材料（录音、聊天截图）的真实性。如认为是伪造证据材料的，欢迎提取资料去做鉴定。我就有这样的底气告诉你！

对于那些“相关人士”来说，认为我可以通过合法途径解决的，如果能解决或者说我能接受这种结果，那我就不会在网络上曝光这些事情了。这是我唯一的路径了，这样你还拦着我、封杀我的话。那我就真的没办法，因为有句古话叫“君要臣死，臣不得不死”。我不想像电影里那样要以暴制暴，这样只会我步入歧途。因为我有老婆、孩子，若我还是当年那样，我会让他们付出代价。泥人还有三分火气、兔子急了都还咬人，何况我也不是什么善人，如果我撑不住了我也会爆发的。

这个世界上有太多的不公和欺诈，你越老实就会越被欺负。虽然说是法治社会，但对于不懂法的穷人、接受教育比较少的人来说又有什么用，可能违法了或者被侵权都不知道。现在的法律可不会因为不懂法就会放过你的，在大多数国家都是这样！

建站的理由！

原本自学前端知识，是为了学习开发小程序。后来因为一场官司让我感到了危机，这让我意识到这个社会的另一面，让我感到前途迷茫、想到因为这个案子带来的后续影响我就头皮发麻。我意识到我需要尽全力去做补救，一审不理想、我就上诉二审，二审还是一样，我便申诉再审，如再审还是一样，我将申诉至最高法院或检察院，或许我还会向相关的部门提交信访材料，去争取我的公道。同时也将我的遭遇尽可能的让更多人知道，让大家知道这种情况是怎么样，让大家别再被骗了。 建设这个网站是为了记录我的这个遭遇，和我的处理方式与最后的结果，同时也记录一些其他的内容。发布相关内容在微博或者其他的网站上，可能连发布的审核都过不了，因为平台不愿意相信我，他们觉得我发布的内容有问题，他们不想承担因我发布的内容带来的一些不可预料的后果。 所以我自己建设网站，自己想发布什么内容就发布什么内容，在不违反相关法律的情况下，我说什么都可以。反正网站是我的，出了什么问题也是找我。

hexo常用命令

新文章或者新的页面

hexo new [layout] <title>
参数 描述
-p, --path 自定义新文章的路径
-r, --replace 如果存在同名文章，将其替换
-s, --slug 文章的 Slug，作为新文章的文件名和发布后的 URL

Hexo 有三种默认布局：post、page 和 draft。

布局 路径 备注
post source/_posts 帖子
page source 页面
draft source/_drafts 草稿

模版（Scaffold）

在新建文章时，Hexo 会根据 scaffolds 文件夹内相对应的文件来建立文件，例如：

$ hexo new photo "My Gallery"

在执行这行指令时，Hexo 会尝试在 scaffolds 文件夹中寻找 photo.md，并根据其内容建立文章，以下是您可以在模版中使用的变量：

变量 描述
layout 布局
title 标题
date 文件建立日期

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量，举例来说：

---
title: Hello World
date: 2013/7/13 20:46:25
---

以下是预先定义的参数，您可在模板中使用这些参数值并加以利用。

参数 描述 默认值
layout 布局 config.default_layout
title 标题 文章的文件名
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于分页） 不适用于page
categories 分类（不适用于分页） 不适用于page
permalink 覆盖文章网址 自定义网站链接
excerpt Page excerpt in plain text. Use this plugin to format the text 纯文本的页面摘录。使用此插件格式化文本
disableNunjucks Disable rendering of Nunjucks tag {{ }}/{% %} and tag plugins when enabled
lang Set the language to override auto-detection ‎继承自‎_config.yml

布局

分类和标签

只有文章支持分类和标签，您可以在 Front-matter 中设置。在其他系统中，分类和标签听起来很接近，但是在 Hexo 中两者有着明显的差别：分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次。

categories:  #分类示例
- Diary
tags:        #标签示例
- PS3
- Games

分类方法的分歧

categories:
  - Diary
  - Life

会使分类Life成为Diary的子分类，而不是并列分类。因此，有必要为您的文章选择尽可能准确的分类。

如果你需要为文章添加多个分类，可以尝试以下 list 中的方法。

categories:
- [Diary, PlayStation]
- [Diary, Games]
- [Life]

此时这篇文章同时包括三个分类： PlayStation 和 Games 分别都是父分类 Diary 的子分类，同时 Life 是一个没有子分类的分类。

引用块

在文章中插入引言，可包含作者、来源和标题。

别号： quote 译文；引用

{% blockquote [作者[, 来源]] [链接] [来源链接标题] %}
content
{% endblockquote %}

generate

$ hexo generate

生成静态文件。

选项 描述
-d, --deploy 文件生成后立即部署网站
-w, --watch 监视文件变动
-b, --bail 生成过程中如果发生任何未处理的异常则抛出异常
-f, --force 强制重新生成文件 Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。 使用该参数的效果接近 hexo clean && hexo generate
-c, --concurrency 最大同时生成文件的数量，默认无限制

该命令可以简写为

$ hexo g

publish

$ hexo publish [layout] <filename>

发表草稿。

server

$ hexo server

启动服务器。默认情况下，访问网址为： http://localhost:4000/。

选项 描述
-p, --port 重设端口
-s, --static 只使用静态文件
-l, --log 启动日记记录，使用覆盖记录格式

deploy

$ hexo deploy

部署网站。

参数 描述
-g, --generate 部署之前预先生成静态文件

该命令可以简写为：

$ hexo d

render

$ hexo render <file1> [file2] ...

渲染文件。

参数 描述
-o, --output 设置输出路径

migrate

$ hexo migrate <type>

从其他博客系统 迁移内容。

clean

$ hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

list

$ hexo list <type>

列出网站资料。

version

$ hexo version

显示 Hexo 版本。

选项

安全模式

$ hexo --safe

在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

调试模式

$ hexo --debug

在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并 提交调试信息到 GitHub。

简洁模式

$ hexo --silent

隐藏终端信息。

自定义配置文件的路径

# 使用 custom.yml 代替默认的 _config.yml
$ hexo server --config custom.yml

# 使用 custom.yml 和 custom2.json，其中 custom2.json 优先级更高
$ hexo generate --config custom.yml,custom2.json,custom3.yml

自定义配置文件的路径，指定这个参数后将不再使用默认的 _config.yml。
你可以使用一个 YAML 或 JSON 文件的路径，也可以使用逗号分隔（无空格）的多个 YAML 或 JSON 文件的路径。例如：

# 使用 custom.yml 代替默认的 _config.yml
$ hexo server --config custom.yml

# 使用 custom.yml, custom2.json 和 custom3.yml，其中 custom3.yml 优先级最高，其次是 custom2.json
$ hexo generate --config custom.yml,custom2.json,custom3.yml

当你指定了多个配置文件以后，Hexo 会按顺序将这部分配置文件合并成一个 _multiconfig.yml。如果遇到重复的配置，排在后面的文件的配置会覆盖排在前面的文件的配置。这个原则适用于任意数量、任意深度的 YAML 和 JSON 文件。

显示草稿

$ hexo --draft

显示 source/_drafts 文件夹中的草稿文章。

自定义 CWD

$ hexo --cwd /path/to/cwd

自定义当前工作目录（Current working directory）的路径。
